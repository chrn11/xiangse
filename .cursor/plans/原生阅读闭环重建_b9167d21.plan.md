# 香色原生阅读闭环重建计划

## 最终目的

让**香色阅读（iOS，`com.appbox.StandarReader`）支持阅读协议（Legado 书源协议）**，且正文渲染**仍由香色原版私有阅读内核**（`TextReadVC3` / `TextRPageContainer` / `TextRPageContainerPage` / `TextRScrollContainer` / `TextReadTV` / `ReadPageModel`）完成。

用户视角的完整闭环：导入阅读协议书源 JSON -> 搜索 -> 详情 -> 目录 -> 点章 -> **看到的是香色原版阅读 UI**（分页/滚动/字体/排版/缓存/进度）-> 翻页/切章/退出再进能恢复 -> 全程感觉不到 Legado 的存在，只觉得"香色能看的书变多了"。

**阅读协议只当书源数据后端，香色原版阅读器是宿主。** 不是用 Legado 自己的阅读器，更不是 overlay/probe 冒充上屏。

## 已冻结决策

- 用 **revert 提交**把 `main` 恢复到 `575864f` 的行为基线；不 hard reset、不改写历史。理由：保留可审计的失败路径，便于事后复盘"为什么硬拼私有 UI 行不通"。
- overlay 仅允许 Debug IPA 使用；Release 禁止 overlay，原生绑定失败时明确 fail-open，不冒充阅读成功。理由：overlay 是诊断工具不是产品形态，Release 出货必须让用户看到真实阅读体验或真实失败。
- 最终范围是完整原生阅读闭环，按硬门禁依次完成：对象/方法证据 -> 第一章上屏 -> 分页切章 -> 缓存进度 -> 滚动模式。理由：分阶段门禁防止"假上屏"蒙混过关，每个阶段都有可验证的退出条件。
- 第一证据源是"原版正常 TXT 阅读 vs Legado 阅读"的同版本差分，不再以全局 selector 字符串邻近作为类归属。理由：字符串邻近是搜索手段不是证据，`b492825` 已证明此路径产生 `sig=6`。
- 最终接入缝是：恢复原版 `loadCurCp` 生命周期，只替换正文数据/缓存提供边界；禁止手工构造私有阅读 UI。理由：原版 UI 生命周期是为原版书源设计的自洽系统，强行拼接会撞 UIKit 层级断言。
- 一个生产代码集成代理；其余代理只产证据或工具，独立分支/worktree，禁止同时直接改 `main`。理由：多代理并发改生产阅读链会导致证据无法归因。
- 先固定横向分页模式，完整过关后再单独做滚动模式。理由：两种模式类归属不同（`TextRPageContainer` vs `TextRScrollContainer`），不能复用假设。
- 新增仅手动触发的 `reader-forensics.yml`，同时产 baseline-debug 和 Legado-debug 身份一致的取证包。理由：自动触发会污染 CI 产物流，且取证包体积大。

## 全局硬规则

1. **禁止生产代码调用/写入 `setPageModel:`、`_textViewL`、`_pageModel`**；禁止 `object_setIvar` 拼私有 UI；禁止手工 `alloc TextReadTV/container`。判定：fixture gate 扫描生产 Hooks 源码，命中任一即 CI 失败。反例：`b492825` 的 `object_setIvar(..., _textViewL, ...)` 直接导致 `sig=6`。
2. **未取得"准确类 + selector + type encoding + IMP/偏移 + 基线调用时序"前，集成代理不得调用任何新私有 API。** 判定：每个新调用的 selector 必须在 `method-map.json` 有 `confidence: confirmed` 条目，且 `evidence_reference` 指向真实 method list 或反编译 xref，而非字符串邻近。
3. **overlay、accessibility probe、`tvHasNeedleProbeOnly`、CI 绿都不算原生上屏。** 判定：`passed` 必须有原生 dump 证据（`pageContainerA` 非空且由原版创建、`nativePaged=1`），OCR 命中"萧炎"且 bounding box 在正文区域。
4. **每个实验提交只验证一个假设；失败即回退该提交，不在失败代码上继续叠补丁。** 判定：commit message 必须含"假设 X"标记；失败假设必须 `Revert "..."` 提交，新假设不得在 revert 前的代码上继续。允许：有指令级杀因取证支撑的剥洋葱式试错（A->J 模式），但每个新假设须引用前次失败的取证分析。
5. **每份真机证据必须带：Git SHA、CI run ID、variant、基线 IPA SHA、LegadoBridge SHA、LegadoBridgeDebug SHA、安装时间和设备 bundle/version。** 判定：devkit `accept` 输出 JSON 缺任一字段即失败；`--expected-sha/--expected-run` 不匹配时拒绝操作真机。
6. **原始 dump、正文、截图留在 ignored 的 `fixtures/_devkit/`；仓库只提交脱敏后的 method map、diff 摘要和自动化脚本。** 判定：`.gitignore` 须覆盖 `fixtures/_devkit/`、`fixtures/_accept_*`（除合同脚本）、`fixtures/_diag_*`、`fixtures/_probe_*`、`fixtures/_tmp_*`、`fixtures/_accept_result_*`、`fixtures/_accept_screenshot*`、`fixtures/_fresh_trace_*`、`fixtures/_install_*`、`fixtures/_peek_*`、`fixtures/_shot_*`。当前工作区 150+ 未 ignore 临时文件违反本条。**执行机制（2026-07-20 修订）**：每个阶段门禁检查前，`git status --porcelain` 未跟踪临时文件必须 < 20；`.gitignore` 模式必须同时覆盖文件与目录（禁用仅目录的尾斜杠模式，已证实漏掉全部文件形态的临时证据，恶化至 260+）；超限时先清理再验收。

### 工厂路径不兼容降级（阶段 4 分支）

当原版工厂路径在 Legado 父 VC 层级下被证伪时，允许集成代理从 `onReset` 触发点降级到 `loadCurCp` 数据入口接入。本分支不是替代阶段 4 主路径，而是其受控分支；触发与执行须严格按以下条款。

**触发判定（同时满足才算证伪）**：
- 真机出现 `NSInternalInconsistencyException` / `EXC_CRASH` / 回 SpringBoard，且崩点位于原版工厂路径同步调用链内（如 `onReset` -> `pageContainer` getter -> `addChildViewController:` -> `insertSubview:atIndex:`）。
- 已有指令级反汇编定位到具体杀点地址（如 `0x10006697c addChildViewController:`），且通过对照实验证明工厂路径在某种 catalog/window/appear 组合下必杀、另一种组合下安全（如 C `cat=0` ORIG_OK vs D `cat=2` 杀）。
- 排除数据问题：偶尔崩溃、Legado 正文缺失、网络失败、mock 源错误不算工厂不兼容，须先修数据再判定。

**允许的降级路径**：
- 入口从 `onReset` 触发点改为 `loadCurCp` 数据入口；Bridge 不再主动 fire 无参 `onResetContentNotify` ORIG。
- Bridge 只在数据边界（章节模型/缓存/正文字符串）注入 Legado 正文，让原版自己走完整生命周期（工厂 + `addChild` + `insertSubview` + 首次绘制）。
- 若原版 `loadCurCp` 自身需要触发 `onReset`，须由原版 appear/didLoad/通知链自然触发，不由 Bridge 主动调用。
- 临时桥接代码（如假设 J 的全局 swizzle `addChildViewController:`/`insertSubview:atIndex:` defer flush）在降级路径验收通过后必须拆除，不得残留。

## 验收环境与观测可信性手册（2026-07-22 新增）

本手册是任务卡 5/6/7 所有真机轮的**强制前置与判读规则**。历史教训：6A 至少 3 轮"失败"实为环境/流程问题而非代码问题；观测源混用导致至少 2 次误判（KVC 路径误报、RecordQuiet 静默）。

### 一、每轮真机前置检查单（缺一即停，不计入假设成败）

1. **mock 可达**：`curl http://<mock>:8765/legado-local-mock.runtime.json` 返回 200；本机 mock 由 `fixtures/serve_local_mock.py` 提供，挂了就地重启。
2. **MCP 健康**：`GET /health` 返回 ios-mcp 版本；设备 `wake_and_home` 后 `locked=false screen_on=true`。锁屏时 uninstall/install 被 blocked，产生"假失败"（2026-07-22 实测）。
3. **同 SHA manifest**：设备上 App 的 `Documents/reader-build-manifest.json` 的 git_commit/github_run_id 与本轮目标 SHA 一致；不一致禁止判读任何探针。
4. **流程 alert 全清**：import 后「导入 N 个书源」→「好」必须点掉（t+1s 即弹）。未点 = 书源未入库 = 目录请求报「未找到 Legado 书源」= awaitingCatalog 永久卡死。
5. **deferred 锁清理**：awaitingCatalog 卡死后，sameBook 的后续 nativeRead 直接 return（`LBOpenNativeChapterAtIndex` 8273 分支，不再发目录请求）；解锁方式只有杀 App 重启（内存态）或换书。验收脚本 nativeRead 前必须 kill+relaunch。

### 二、观测源可信性分级（判读前必须先查此表）

| 观测源 | 文件 | 可信级 | 已知坑 |
|---|---|---|---|
| LBABSyncProbe | legado_ab_probe.txt | **高**（无 quiet 门控，POSIX 直写 fsync） | 无 |

main queue 饿死的 fingerprint（BC10 轮 confirmed）：
1. 同步 main=1 探针序列完整（invoke 尾部数条），随后 main 再无任何动作；
2. 所有 dispatch_async(main) 异步块缺席（drain pulse、qf_dispatch_main_pulse、async_plus0.6s、QF 块）；
3. 无 fault_signal（非 SIGSEGV/SIGABRT）；
4. 进程约 1s 后消失/重启（SIGKILL 类，可能无 .ips）。

出现 1+2+3 即判 **main queue 饿死**，禁止归因为"QF 未投递"；下一步是定位 main 停滞点（RunLoop mode + 调用者返回点探针），不是改 CB/QF 链。

### 四、假设 commit 标准结构（违反即门禁拒）

每个 fix(XX) commit message 必须四段：
1. **假设**：一句话 + 置信度（confirmed/probable）+ 证据引用（文档§节/探针行）；
2. **改动**：单一机制，禁多机制混合；
3. **判定字段**：真机轮里看哪个文件的哪个标签，什么值算过/证伪；
4. **证伪联动**：若证伪，revert 本 commit 或改默认禁用（规则 12）。
